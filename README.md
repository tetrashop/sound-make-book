🎯 معرفی پروژه

sound-make-book یک وب‌اپلیکیشن است که متن ورودی کاربر را با استفاده از موتور eSpeak به کتاب صوتی تبدیل می‌کند. این پروژه با هدف ارائه یک راه‌حل آفلاین، رایگان و فارسی برای تولید کتاب‌های صوتی توسعه یافته است.

پروژه در دو نسخه ارائه می‌شود:

1. نسخه محلی (کامل): با تمام ۹ قابلیت، اجرا روی دستگاه شخصی با Termux یا لینوکس.
2. نسخه آنلاین (معرفی): مستقر روی Vercel، فقط برای نمایش پیام راهنما (به دلیل محدودیت محیط Serverless).

✨ قابلیت‌های پروژه (۹ قابلیت)

# قابلیت توضیح وضعیت پیاده‌سازی
1 تشخیص خودکار زبان متن تشخیص زبان ورودی (فارسی/انگلیسی) با استفاده از کتابخانه franc ✅ کامل (محلی) / ⚠️ شبیه‌سازی (آنلاین)
2 پشتیبانی از فرمت‌های مختلف ورودی آپلود فایل‌های PDF، DOCX، TXT و استخراج متن ⚠️ در برنامه (نیاز به کتابخانه‌های اضافی)
3 تنظیم چندصدایی تقسیم متن به بخش‌ها و اختصاص صدای متفاوت به هر بخش ✅ کامل (محلی)
4 ذخیره پروژه‌ها در فضای ابری اتصال به Google Drive و ذخیره فایل‌های صوتی ⚠️ شبیه‌سازی (نیاز به OAuth)
5 موسیقی پس‌زمینه افزودن موسیقی متن (پیانو، سینماتیک) با کنترل حجم ✅ کامل (محلی)
6 استخراج متن از تصاویر (OCR) خواندن متن از تصاویر اسکن‌شده با Tesseract.js ⚠️ شبیه‌سازی (آنلاین) / نیاز به نصب
7 پیش‌نمایش صدا پخش ۱۰ ثانیه اول قبل از تولید کامل ✅ کامل (محلی)
8 تنظیمات پیشرفته صدا کنترل سرعت گفتار، زیروبمی صدا و فاصله بین کلمات ✅ کامل (محلی)
9 داشبورد مدیریت پروژه‌ها مشاهده، دانلود و حذف پروژه‌های قبلی ✅ کامل (محلی)

وضعیت‌ها:

· ✅ کامل: پیاده‌سازی شده و در نسخه محلی کار می‌کند.
· ⚠️ شبیه‌سازی: در نسخه آنلاین با پاسخ‌های نمونه کار می‌کند (بدون پردازش واقعی).
· ⏳ در برنامه: نیاز به توسعه بیشتر.

🧠 معماری فنی

نسخه محلی (کامل)

· زبان: Node.js (بدون فریم‌ورک، با ماژول‌های داخلی http, fs, path)
· موتور تبدیل متن به صدا: eSpeak (نصب جداگانه روی سیستم)
· پردازش صدا: ffmpeg (تبدیل WAV به MP3)
· فرانت‌اند: HTML5, CSS3, JavaScript (ساده)
· ذخیره‌سازی پروژه‌ها: فایل JSON در پوشه projects/
· فایل‌های صوتی: ذخیره در پوشه audio-cache/

نسخه آنلاین (معرفی)

· پلتفرم: Vercel (Serverless Functions)
· محدودیت‌ها: عدم دسترسی به eSpeak و ffmpeg → فقط پیام راهنما نمایش داده می‌شود.
· هدف: معرفی پروژه و هدایت کاربران به اجرای محلی.

🚀 نصب و راه‌اندازی

اجرای محلی (با قابلیت‌های کامل)

۱. نصب پیش‌نیازها

در Termux (اندروید):

```bash
pkg update && pkg upgrade -y
pkg install nodejs espeak ffmpeg git -y
```

در اوبونتو/دبیان:

```bash
sudo apt update
sudo apt install nodejs npm espeak ffmpeg git -y
```

۲. کلون کردن مخزن

```bash
git clone https://github.com/tetrashop/sound-make-book.git
cd sound-make-book
```

۳. اجرای سرور

```bash
node api/index.js
```

۴. باز کردن در مرورگر

```bash
termux-open-url http://localhost:3000   # در Termux
# یا دستی: http://localhost:3000
```

اکنون صفحه با ۵ تب و قابلیت‌های کامل نمایش داده می‌شود.

استقرار روی Vercel (نسخه معرفی)

```bash
# نصب Vercel CLI
npm install -g vercel

# ورود به حساب (ممکن است نیاز به VPN باشد)
vercel login

# استقرار
vercel --prod
```

یا از طریق وب‌سایت Vercel (با اتصال به گیت‌هاب).

🌐 API Reference

همه endpointها در مسیر /api قرار دارند و پاسخ‌ها به صورت JSON هستند.

Method Endpoint توضیح نمونه درخواست
GET /api/health وضعیت سلامت سرور -
POST /api/detect-language تشخیص زبان متن {"text":"سلام"}
POST /api/ocr استخراج متن از تصویر (multipart/form-data)
POST /api/preview پیش‌نمایش ۱۰ ثانیه {"text":"...", "voice":"fa"}
POST /api/generate-multi-voice تولید با چند صدا {"segments":[{"text":"...","voice":"fa"}]}
POST /api/generate-advanced تولید با تنظیمات پیشرفته {"text":"...","speed":1.2,"pitch":5}
GET /api/projects لیست پروژه‌ها -
POST /api/projects ذخیره پروژه جدید {"name":"...","audioUrl":"..."}
DELETE /api/projects/:id حذف پروژه -

توجه: در نسخه آنلاین، همه endpointها به جز / و /api/health خطای ۴۰۴ برمی‌گردانند.

🔧 عیب‌یابی مشکلات رایج

۱. خطای EADDRINUSE: address already in use

علت: پورت ۳۰۰۰ توسط برنامه دیگری اشغال شده است.
راه‌حل:

```bash
lsof -i :3000   # پیدا کردن PID
kill -9 <PID>   # کشتن فرآیند
# یا تغییر پورت در فایل api/index.js
```

۲. خطای espeak: command not found

علت: eSpeak نصب نیست.
راه‌حل: pkg install espeak (در Termux) یا sudo apt install espeak (در لینوکس).

۳. خطای ۵۰۰ در Vercel

علت: تلاش برای اجرای eSpeak در محیط Serverless.
راه‌حل: از قبل پیش‌بینی شده و پروژه در Vercel فقط پیام راهنما نمایش می‌دهد. برای قابلیت‌های کامل، محلی اجرا کنید.

۴. فایل‌های CSS/JS لود نمی‌شوند

علت: مسیردهی اشتباه یا پوشه‌های styles و scripts وجود ندارند.
راه‌حل:

```bash
mkdir -p public/styles public/scripts
# و مطمئن شوید فایل‌ها در آنجا هستند
```

🗣️ رفع مشکل پشتیبانی از زبان فارسی در نسخه اندروید

اگر در نسخه اندروید با پیام «زبان فارسی پشتیبانی نمی‌شود» مواجه شدید، به این معناست که بسته‌ی صوتی زبان فارسی روی دستگاه شما نصب نیست. برای رفع این مشکل، مراحل زیر را انجام دهید:

📥 دانلود و نصب بسته صوتی فارسی

1. مخزن رسمی بسته صوتی پروژه را باز کنید:
       tetrashop/tts-persian-packages
2. به بخش Releases بروید و آخرین نسخه (espeak-persian-data.tar.gz) را دانلود کنید.
3. فایل دانلود شده را در مسیر زیر در گوشی خود از حالت فشرده خارج کنید:
   ```
   /data/data/com.termux/files/usr/share/
   ```
   (برای این کار می‌توانید از یک برنامه مدیریت فایل مانند CX File Explorer استفاده کنید.)
4. پس از کپی کردن فایل‌ها، برنامه sound-make-book را مجدداً راه‌اندازی کنید. حالا موتور TTS قادر به تولید صدای فارسی خواهد بود.

نکته: این بسته صوتی بر پایه موتور متن‌باز eSpeak تهیه شده و کاملاً رایگان و آفلاین است.

🤝 توسعه و مشارکت

اگر ایده یا بهبودی دارید، خوشحال می‌شویم:

2. یک شاخه جدید (git checkout -b feature/amazing-feature) ایجاد کنید.
